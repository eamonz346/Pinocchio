# Pinocchio 🤥

👋 Hello！！！

这是我的项目：Pinocchio。

Pinocchio 是一个以 DeepSeek 模型为核心的 AI 工作台。
不是单纯的聊天窗口，主要有三个核心模块：chat、plan、canvas。
如果是下达需要交付产出的指令，理想逻辑是AI先思考，然后出计划，在plan卡片展示，用户同意后开始执行，并在canvas展示。  
`然而很可惜，目前这个工作流并不稳定，输出质量也不太行` 
`这能算一个最小 MVP 可行性验证，不是成熟产品。经我亲测，并不好用` 😭

## 目前能做什么

- 💬 **Chat**：基于 DeepSeek 的对话工作台。
- 🧠 **Thinking**：支持切换深度思考模式和思考努力程度。
- 🌐 **Web**：支持联网搜索和网页内容读取。
- 📁 **Files**：支持上传并读取 txt、md、json、csv、pdf 等文件。
- 🖼️ **Canvas**：可以把长文、方案、代码或其他内容放到右侧画布里编辑和预览。
- 📊 **PPT**：尝试通过网页渲染的方式生成演示内容；当然了并不只是PPT，还可以展示一些研究数据、产品报告等。

## 真实体验

⚠️ 联网搜索很多时候会卡在人机验证，文件读取纯靠运气。  

🧠 深度思考还行，deepseek官方本来就支持不同思考努力程度，只是做了一个切换按钮。  

💔 PPT 功能目前非常不建议大家用。讲道理，用起来应该是：用户下发任务后，Pinocchio 会先生成一个计划，然后再开始做 PPT。但是经常plan卡片弹不出来，`反正目前plan功能其实是乱做的`，而且目前不论是内容质量，还是最终渲染成果，效果都还不好。

## 为什么做这个

最近刚好了解到agent这个概念，恰好Claude code、codex等工具也越来越牛逼。  

本人其实也是deepseek的死忠粉，但由于deepseek网页版的功能实在有限，最近v4出来，我觉得这个模型应该有一套更好的挂载功能 
所以就开始做这个工具，主要就是集成聊天、搜索、读文件、写长文、做计划、产出可视化内容这些功能。
`但是难度比我想得更大，而且大得多` 😭

我其实并不懂什么技术，只能靠 AI 。然而AI 给我的东西，总喜欢掺点史。因为只有 UI 我能直接验证质量，所以目前这套 UI 算是勉强合格；其他功能还需要继续打磨。
做到现在，我发现AI确实能帮我们做很多事情，尤其是很多学习起来有门槛、有难度的 事情。很多时候不是AI输出的质量有问题，而是不符合我们的预期。我拿codex做东西的时候，复杂的任务我都开计划模式，但经常codex问我的问题我不懂，也不知道该怎么做选择。很多时候稀里糊涂的就把东西做完了，哪里有问题我都只能交给codex去改，但往往它又不能找到真正的痛点，如此循环往复，离最初的目标越来越远。

后面一阶段的想法是：
📘 自己学代码，至少能读懂我现在用的Typescript。
📕 学习毛选，深度研究功能是直接把qiushi_skills喂给AI做的，但是里面有什么我也不知道。
📗 学习怎么做PPT、html、css、学排版、学配色。
🔍 这个项目让AI抄了很多开源项目，但这些项目我基本上都没自己看过，后面我决定自己看完吸收，再来做优化。
目前只想这么多了，就这些东西也够我学很久了😭

我是第一次做项目，也是第一次开源，readme写的是有一点烂`确实是很烂`，不过这都是我对这个项目所有的理解了，希望大家理解💚也希望大家能帮我一起做这个项目！！！
